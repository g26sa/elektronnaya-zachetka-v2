# Развёртывание для демонстрации (3-4 дня)

Приложение упаковано в Docker (Next.js + SQL Server в отдельных контейнерах).
На сервере достаточно установить Docker — всё остальное соберётся само.

## 1. Аренда облачного сервера на Timeweb Cloud

1. Откройте **cloud.timeweb.com** и зарегистрируйтесь (email + телефон для
   подтверждения).
2. Пополните баланс банковской картой. Для конфигурации ниже на 3-4 дня
   почасовой тарификации хватит **300-500 руб**.
3. В личном кабинете откройте раздел **«Облачные серверы»** →
   **«Создать сервер»**.
4. На странице создания сервера:
   - **Локация** — любой российский регион (например, Москва или
     Санкт-Петербург).
   - **Образ ОС** — вкладка «Linux», выберите **Ubuntu 22.04 LTS**
     (или 24.04).
   - **Конфигурация** — в конфигураторе вручную задайте:
     - **vCPU**: 4
     - **RAM**: 8 GB
     - **Диск (NVMe)**: 40 GB
     (SQL Server Express требует ~2 GB сам по себе, плюс приложение и
     Chromium для генерации PDF — берите с запасом).
   - **Тип тарификации** — переключите на **почасовую** (не помесячную),
     чтобы платить только за реально работающие часы.
   - **Способ аутентификации** — выберите «Пароль» (Timeweb сгенерирует
     пароль root и покажет/пришлёт его) или «SSH-ключ», если он у вас уже
     есть.
5. Нажмите **«Создать»**. Через 1-2 минуты сервер появится в списке со
   статусом «Активен» и публичным **IP-адресом** — он понадобится далее.

При такой конфигурации с почасовой тарификацией стоимость обычно
~1.5-3 руб/час, то есть за 3-4 дня — порядка 150-300 руб, если выключать
сервер в моменты, когда демонстрация не идёт (выключенный сервер не
тарифицируется или тарифицируется по минимуму — уточните в разделе
«Тарифы» вашего сервера).

## 2. Подключение и установка Docker

Подключитесь по SSH с локального компьютера (PowerShell):

```powershell
ssh root@<IP_сервера>
```

Введите пароль root, который выдал Timeweb (при первом входе он может
попросить сменить пароль — задайте новый и запомните его).

Установите Docker:

```bash
curl -fsSL https://get.docker.com | sh
```

## 3. Загрузка проекта на сервер

> **Важно:** новые файлы для Docker (Dockerfile, docker-compose.yml,
> .env.docker.example, docker/migrate.sh, обновлённый next.config.mjs и
> т.д.) пока существуют только локально. Перед клонированием на сервере
> закоммитьте их и сделайте `git push` в репозиторий — иначе сервер
> склонирует версию без Docker-конфигурации.

Самый простой способ — через git:

```bash
git clone https://github.com/g26sa/elektronnaya-zachetka-v2.git elektronnaya-zachetka
cd elektronnaya-zachetka
```

Если репозиторий приватный, git запросит логин/пароль — вместо пароля
используйте personal access token GitHub (Settings → Developer settings →
Personal access tokens).

Либо скопируйте архив проекта через `scp` с локального компьютера:

Выполняется на локальном компьютере в PowerShell, **не** на сервере:

```powershell
scp -r "C:\Users\Самира\Desktop\elektronnaya-zachetka" root@<IP_сервера>:/root/
```

## 4. Настройка окружения

На сервере:

```bash
cp .env.docker.example .env.docker
nano .env.docker
```

Заполните:
- `DB_PASSWORD` — пароль для SQL Server (минимум 8 символов, должны быть
  заглавные, строчные буквы и цифры/спецсимволы)
- `JWT_SECRET` — сгенерировать командой `openssl rand -base64 48`
- `APP_URL` — `http://<IP_сервера>` (без слеша на конце)
- `SEED=1` — чтобы база заполнилась демо-данными при первом запуске
- SMTP-поля можно оставить пустыми — письма для сброса пароля будут
  выводиться в лог контейнера `app`

## 5. Запуск

```bash
docker compose --env-file .env.docker up -d --build
```

Сборка образа (включает `npm ci` и `next build`) занимает несколько минут.
Проверить ход миграции/сидинга базы:

```bash
docker compose logs -f migrate
```

Когда появится строка `==> Миграция завершена.`, приложение доступно по
адресу `http://<IP_сервера>` (порт 80).

## 6. Вход в демо-версию

После сидинга (`SEED=1`) в базе будут демо-пользователи (пароль для всех —
`demo1234`):

- `head@college.local` — заведующий
- `teacher1@college.local`, `teacher2@college.local`, `teacher3@college.local` — преподаватели
- `student1@college.local`, `student2@college.local`, `student3@college.local` — студенты

**Перед защитой смените эти пароли** или создайте отдельного пользователя
с реальными данными через интерфейс администрирования.

## 7. Открыть порт в файрволе (если включён UFW)

```bash
ufw allow 22/tcp
ufw allow 80/tcp
ufw enable
```

## 8. Полезные команды

```bash
# логи приложения
docker compose logs -f app

# перезапуск после изменений в коде
docker compose --env-file .env.docker up -d --build

# остановить всё (данные в volume сохраняются)
docker compose down

# полностью удалить вместе с данными БД и загруженными файлами
docker compose down -v
```

## 9. После защиты

Чтобы не платить за лишнее время — остановите/удалите сервер у провайдера
(или хотя бы выполните `docker compose down`, чтобы освободить ресурсы перед
выключением VPS).
